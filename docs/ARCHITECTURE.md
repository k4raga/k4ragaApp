# Architecture

## Домены

Приложение делится на два смысловых слоя:

- `hub` — разводная на основном домене;
- `cs training` — конструктор и прохождение тренировки на `cs`-поддомене.

## Маршруты

- `/`
  - на `k4raga.ru` рендерит обычную разводную;
  - на `cs.k4raga.ru` рендерит конструктор тренировки.
- `/builder`
  - прямой route к тому же конструктору.
- `/training`
  - страница прохождения активной тренировки.

Host-aware поведение собрано в `app/page.jsx`.

## Модель данных

Источник правды по блокам и фокусам:

- `lib/training-data.js`

Там лежат:

- список блоков;
- список фокусов или групп фокусов;
- обзорные блоки для landing;
- генерация начального состояния;
- случайный выбор фокусов.

## Слой состояния

Логика состояния собрана в:

- `lib/training-state.js`

Файл отвечает за:

- чтение из `localStorage`;
- запись в `localStorage`;
- построение активной тренировки;
- сброс чеков;
- реролл фокусов;
- переключение блока;
- переключение фокуса.

Это нужно, чтобы `builder` и `training` не дублировали одинаковые мутации.

## UI

- `components/BuilderScreen` — вся сборка и фильтрация сетки;
- `app/training/page.jsx` — только прохождение выбранной сетки;
- `components/Topbar` — навигация, зависящая от host;
- `components/SiteChrome` — общий фон и шапка.

## Текущий принцип тренировки

- один блок = один слот на тренировке;
- у блока есть пул фокусов;
- если активны несколько фокусов, в текущую сессию выбирается один случайный;
- кнопка `Обновить` на странице тренировки делает новый случайный набор и сбрасывает прогресс.
